# Worked Examples

Redacted, fictional example proposals demonstrating the shape of a completed workspace for each major proposal type. Read these to see how the pieces fit together before you start a real proposal.

## Index

| Example | Type | What it demonstrates |
|---|---|---|
| [cso-brief-acme/](cso-brief-acme/) | `cso-brief` | Type registry dispatch, compliance matrix with eval criteria, all 4 writing patterns, ROM pricing artifact, lightweight Gold Team scorecard |

## How to use these

- **Read top-down.** Start with the example's own `README.md` for the scenario and a guided tour of the files.
- **Don't copy blindly.** Examples use fictional customers, agencies, and company names. Adapt the *patterns* they illustrate, not the *content*.
- **Focus on the working/ files.** Those show how `proposal-type.md`, `proposal-plan.md`, `compliance-matrix.md`, and `activity.md` evolve across a proposal lifecycle — that's the most reusable knowledge.

## Disclaimer

All customers, agencies, contracts, and technical specifics in these examples are fictional. Any resemblance to real programs is coincidental. These exist to teach workflow, not to represent real opportunities.
