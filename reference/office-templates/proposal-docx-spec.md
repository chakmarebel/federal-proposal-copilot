# Proposal docx Spec

Specification for Word documents produced by `/export-proposal` from `drafts/*.md`.

## Base template precedence

1. `my-company/templates/proposal.docx` (user-branded — preferred)
2. `my-company/templates/<type_id>.docx` (type-specific user template)
3. Default (generated by `anthropic-skills:docx` using the conventions below)

## Document structure

### Cover Page (first page)

- Company logo (top-left, pulled from `my-company/brand-palette.md` or `my-company/assets/logo.png` if present)
- Proposal title (centered, 24pt bold)
- Solicitation reference (centered, 14pt)
- Offeror block: CAGE, UEI, NAICS, size status (from `my-company/company-description.md`)
- Date (centered, 11pt)
- Distribution statement (bottom, per `reference/distribution-statements.md` if applicable)
- Volume designator if this is a multi-volume proposal (Technical Vol / Management Vol / etc.)

### Table of Contents (auto-generated)

Word-native TOC field that updates from heading styles. Page numbers start on the page after TOC (page i for TOC, page 1 for body).

### Header / Footer

**Header (every body page):**
- Left: Proposal short title (from `working/proposal-brief.md`)
- Right: Volume name

**Footer (every body page):**
- Left: Company name + CAGE
- Center: Distribution statement (abbreviated, if applicable)
- Right: "Page X of Y"

### Body styles

Use Word heading styles (not manual formatting) so the TOC auto-generates:

| Markdown | Word style | Typography |
|---|---|---|
| `# Heading 1` | Heading 1 | 18pt bold, company accent color, 12pt space before |
| `## Heading 2` | Heading 2 | 14pt bold, company accent color, 6pt space before |
| `### Heading 3` | Heading 3 | 12pt bold italic, 6pt space before |
| Body paragraphs | Normal | 11pt, 1.15 line spacing, 6pt after |
| Block quotes | Quote | 10pt italic, indented |
| Tables | Table Grid | 10pt, header row shaded |
| Code / monospace | (rare in proposals) | 10pt Consolas |

### Figures

Every `See Figure N` or graphics reference in the markdown should become:

1. The embedded PNG from `final/graphics-png/figN-*.png` (centered, sized to fit within margins)
2. Below the image: action caption in "Caption" style — `Figure N. [Title].` in bold, followed by the caption body in 10pt italic (pulled from `working/graphics-brief.md`)

### Tables

Markdown tables convert to Word Table Grid style. For tables that are too wide to fit portrait, switch that section to landscape (Word supports per-section orientation).

### Page limits

Per `page_target` in `working/proposal-type.md`. The export skill reports estimated page count after export; if over target, surface the warning for the user to trim in markdown.

## Font choices

- **Default:** Calibri 11pt body / Calibri 14-18pt headings (Word's modern default)
- **If `my-company/templates/proposal.docx` defines different fonts:** use those (user has branded the template)
- **Government-preferred alternative:** Times New Roman 12pt body (some solicitations specify 12pt minimum — respect that)

## What NOT to include

- **Footnotes** — rare in federal proposals. Use parenthetical references or Endnotes sections instead.
- **Track changes from drafting** — accept all before export.
- **Comments** — accept or delete before export.
- **Hyperlinks to external URLs** — generally avoid; evaluators may read printed.
- **Colored text for emphasis** — use bold/italic instead. Color is OK in graphics and in heading styles; not in body text.

## Multi-volume proposals

For `far-rfp` and similar, produce separate .docx files per volume:
- `final/docx/technical-volume.docx`
- `final/docx/management-volume.docx`
- `final/docx/past-performance-volume.docx`
- `final/docx/cost-volume.docx` (narrative; xlsx appendix separate)

Each volume has its own cover page, TOC, and page numbering.

## Final step (user-driven)

After the .docx is produced, the user:
1. Opens in Word
2. Reviews formatting (spot-check page count, figure legibility, table overflow)
3. File → Save As → PDF → saves to `final/pdf/`

**Do not attempt to produce PDF from within this skill.** Word's native PDF export is the only format-preserving option.
