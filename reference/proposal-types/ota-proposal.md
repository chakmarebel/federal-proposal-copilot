# OTA Full Proposal

## Evaluator mindset
"Can this team deliver the prototype against the Statement of Objectives, and can we structure a deal that includes follow-on production?"

## Required artifacts
- Statement of Objectives → approach mapping (SOO-to-section crosswalk)
- Milestone-payment schedule (deliverable + payment + acceptance criteria per milestone)
- Prototype project description with clear exit criteria
- Follow-on production language (10 USC 4022(f))
- Data rights assertions (noncommercial technical data, computer software)
- Non-traditional contributor justification
- Relevant experience (no formal PPQs)

## Common pitfalls
- Submitting a FAR cost volume instead of a milestone schedule
- Forgetting follow-on production language (you can't add it later)
- Over-asserting data rights without specific technical data markings
- Generic SOO response that doesn't crosswalk to deliverables
