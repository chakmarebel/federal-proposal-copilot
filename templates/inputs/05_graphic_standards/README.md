# Graphic Standards (Inputs)

Visual standards, brand references, and reusable HTML/SVG templates copied from prior proposals or brand guidelines. These are **inputs** to `/proposal-graphics` — they help the skill produce on-brand graphics faster.

## What goes here

- Brand palette references (if not already in `my-company/brand-palette.md`)
- HTML graphic templates from past winning proposals (three-tier architecture, capability matrix, timeline patterns, etc.)
- Example figures you want the new proposal to match visually
- Approved logos, icon sets, or visual standards documents

## What does NOT go here

- **Rendered graphics for this proposal** — those go in the top-level `graphics/` directory (they're outputs, not inputs)
- **Generated specs or wireframes** — those go in `working/graphics-brief.md`

## How `/proposal-graphics` uses this

When producing graphics for this proposal, `/proposal-graphics` will look here for existing templates to adapt rather than designing from scratch. If a useful pattern exists here, the skill reuses the layout, colors, and typography and replaces only the content.
